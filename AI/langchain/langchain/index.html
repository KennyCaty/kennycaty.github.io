LLM到Langchain

人是如何与LLM交流的？
“请给我写一个笑话”

“1+1=几？”

我们把这个输入给LLM的语句叫做提示词 Prompt （介绍Prompt的概念）

大型语言模型/LLM示例

  • GPT3/3.5/4 （MoE）
  • LLaMA（Meta数据泄露，Meta的开源语言模型）
  • ChatGLM（中文语料训练的开源模型，实际中文能力可能低于ChatGPT，因为GPT规模更大）

可以自行了解一下MoE和Meta数据泄露

大型语言模型的发展

  • GPT3， June 11， 2020发布
  • 2022.11 OpenAI发布GPT3.5
  • 12月，互联网热度爆发
  • 12月，Langchain第一版发布

为什么Langchain紧跟着就发布了？ 我们先了解一下OpenAI的API长什么样子，有啥局限性

  • GPT API模板（抽象）

temperature代表LLM的发散能力，值越高，LLM的回答想象力（不确定性）就越大，值越低，就会越接近自己训练的内容，不确定性越低

这种API call不能很好的解决某些问题：

  • 如何格式化输出，比如json等
  • 如何输入很长的文本（token有限，你怎么让它总结一本书的内容？）
  • 如何多次进行API调用（几个连续的问题，上一个的输出是下一个的输入，或者是调用外部API）
  • 如何让api能够调用外部的服务
  • 如何进行标准化开发
  • 如何快速切换模型
  • …

Langchain诞生，并解决这些问题

Langchain🦜️

LangChain是一个用于开发由语言模型驱动的应用程序的框架。Langchain官网（python）

langchain相当于一个中间工具，用户先跟Langchain交流，Langchain再跟LLM交流

传统API调用

graph LR
    A(User)-->B(LLM)

Langchain

graph LR
    A(user)-->B(Langchain)-->C(LLM)

Langchain的设计思路：

  • 抽象化
  • 模块化

把和大型语言模型交互的流程抽象出来，变成一个一个的小模块，方便更好的进行调用。通过把这些小模块串联（chain）起来，实现更多更复杂的功能。

Chain

langchain中的链式结构

为什么需要链式结构

  • 方便的连接多个（LLM/其他）模块，使得功能更加丰富

如何避免重复定义功能相似的LLM模块？

  • 功能相似：比如一个给男孩起名，一个给女孩起名，功能都相同只是对象不一样，就不必要定义成两个模块

  • 解决：提示模板（prompt template）

Coding

Agent

为什么需要代理

  • 处理链式模块难以独立处理（LLM不能独立处理）的复杂问题
    • 比如说需要搜索互联网，由于LLM没有接入互联网，不能单独完成此任务，需要接入搜索互联网的API（比如google API）
  • 动态决策

本质

  • 对于一个任务，运用语言模型来决定完成任务所需的行为以及实施这些行为的顺序

行为方式

  • agent可以使用一系列的预设的工具（tools）
    • 选择工具（包括是否选择）
    • 使用工具
    • 观测并处理工具使用的结果
    • 重复以上步骤

LLM模块

代码：llm_deep.ipynb

大型语言模型交互模块（Model I/O）

这个模块的主要功能就是对大型语言模型进行抽象，使得使用更方便

Prompt

模型交互需要一套交互的”编程方式“——提示（prompt）

这个提示可以包括：

  • 对话角色背景
  • 对任务本身的规范

提示的精确或冗余程度：

  • 越是精确的提示 - 理想的结果
  • 越是冗余的提示 - 稳定的结果

比如：

“给我一个小狗名字” -》“旺财”

“给我一个很土但是听起来很好养活的小狗名字” -》 “狗蛋”

Langchain对提示词也有自己的封装 - 提示模板（prompt templates）

“给我一个很土但是听起来很好养活的 {Object} 名字”

Coding

怎么样使用提示模板呢

提示模板可以包括：

  • 对语言模型的指令

  • 提供一些简单的示例给语言模型从而使模型输出更接近理想的结果

  • 提给语言模型的问题

  • …

注意：提示模板可以接收多个参数，甚至可以不接受参数（就剩下提示词本身）

接收多个参数

什么时候只接受部分参数呢？有时候不能获得所有参数，在链式结构中传递提示模板时，不需要一次性提供所有参数

少样本学习

可以给语言模型在提示词中提供一些例子来规范语言模型的功能

这种提示方式叫做”少样本学习“ （few-shot）

  • 通过少量的样本（案例）让语言模型学会处理某类特定问题
  • 和对语言模型本身的微调(fine-tuning)区分

相当于提供LLM回答问题的套路

举例

  • ”你好吗“ - ”帅哥，我很好“
  • ”今天周几“ - ”帅哥，今天周日“
  • ”天气真好“ - ”帅哥， 今天天气确实不错“
  • ”我放了别人鸽子“ - ？？

相当于我们想让模型学会“帅哥”这个前缀， 它可能回答“帅哥，你真狗 ”

few-shot和fine-tuning区别

  • few-shot给模型提供了一系列如上的例子，并得到一个输出，没有改变模型本身
  • fine-tuning直接影响模型本身，用了大量例子调整模型本身的参数，与老模型是两种不同的模型

利用样本筛选器筛选样本

为什么需要样本选择器(ExampleSelector)

  • 样本数量太多
  • 不是所有样本都能起到提升输出质量的作用
  • 省钱（减少提示词token数量）

SemanticSimilarityExampleSelector 语义相似度筛选器

  • 根据最终提的问题，去寻找样本中与这个问题语义上更接近的样本，把这些样本发送给LLM

此类根据与输入的相似性来选择少数样本。它使用嵌入模型来计算输入和少数样本之间的相似度，并使用向量存储来执行最近邻搜索。

数据连接模块

代码: data_conn.ipynb

数据连接

一些基于大型语言模型的应用经常需要用到模型数据集中没有的数据。针对这一需求，Langchain提供了一系列的工具可以让你从各种数据源中加载新的数据，转换数据，储存数据以及访问数据

  • 文档加载器：从多种不同的数据源加载文档
  • 文档转换器：拆分文档、丢弃冗余文档等
  • 文本embedding模型：将非结构化文本转化为浮点数的列表
  • 向量存储站：存储和搜索embedding数据
  • 检索器：查询向量数据

这一节主要介绍一些概念

数据连接处理流程

langchain在加载数据时，用到的是数据加载器（Document loaders）

  • 输入：各种数据源

文档转换器

Document transformers

一旦加载了文档，通常会希望对其进行转换，从而更好地适应应用场景

最简单的例子是可能希望将长文档拆分为较小的块，以避免大型语言模型对于提示词文本长度的限制

LangChain具有许多内置的文档转换器，可以轻松地拆分、合并、过滤文档。

文本分割器

  • 分割长文本
  • 分割策略：
    • 根据语言相关性把有关联的文本放在同一个分割段中

工作方式

  • 将文本拆分成小的，语义有意义的块（通常是句子）
  • 开始将这些小块组合成较大的块，直到达到一定的体量。
  • 一旦达到该大小，将该块作为独立的文本片段，然后开始创建一个新文本块。
    为了持块之间的连贯性，一般两个文本块之间会有重叠的部分。

文本词嵌入模型

Word Embedding

  • 词嵌入是词语的一种数值化表示方式，一般情况下会将一个词映射到一个高维的向量中（词向量vector)
  • 例子
    -”我”->[0,0.3，-0.1]
    -“你”->[-0.6, 0.12, 0.16]
    -“俺”->[0.01, 0.298，-0.99]
  • 不再考虑单词或者句子，对象变成了高维向量->数字化表达
  • 接近的语义=接近的高维距离
    • 比如 “我” 和 “俺” 两个词的向量距离较近，可以理解为同（近）义词
  • 把文本变成高维向量：嵌入模型
    • openAl
    • Cohere
    • Hugging Face

向量存储

上面提到的词向量，可以将文档转化为以词向量表示的文档，可以用来检索同义词等等，但是当一次使用结束后，下一次在需要检索（使用）这个文档时，需要重新在Embedding一遍吗？

我们可以把它存起来，想用的时候直接拿

实际上就是把词向量存储在数据库里面

  • 嵌入模型导出的向量需要放到一个地方便于存储和查询（检索）
  • 是文本的数据库
    • 文本以嵌入向量的形式存储

存储之后，需要在数据库中进行检索

检索器

  • 检索器是一种根据非结构化查询语句返回对应文档的接口
  • 相比向量存储而言，检索器的定义更加宽泛
    • 向量存储可以看作是一种具备存储功能的检索器实现
    • 检索器不一定需要具备存储向量的功能

Chain

Chain，是Langchain的模块化，抽象化能力

什么是链式结构？

  • 定义提示词
  • 选择llm
  • 运行inferencing
  • 得到结果

有什么用？

为了一个抽象化模块化能力，可扩展可复用性

  • 概念上，链是在规定的标准下，不同模块的组合。
  • 链是一个抽象出来的概念/标准，并不是一定要用链才能完成对LLM的调用。
  • 链提高了模块的标准化，复用性。
  • 链增加了工程的复杂度，冗余度。

链结构分类

  • 基础链结构

    • LLM chain
    • Router chain
    • Sequential chain
    • Transformation chain

  • 应用链实例

    • Document Chains
    • Retrieval QA
